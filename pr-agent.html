What it is

汽车公关的
AI 内容工作台

它围绕真实内容任务来设计，帮助用户把分散的工作串成一条更清楚的流程。

PR Agent 是一个给汽车公关用的内容助手。它可以根据 brief 和资料整理传播方向，生成标题、导语、新闻稿段落和社媒文案；也可以继续处理已有稿件，完成审稿、统一表达和改写。

我想做的是一个真的能放进工作里的工具。用户知道从哪里开始，知道该输入什么，也知道系统正在帮自己完成哪一步。

Workflow

从 brief 到
可交付内容

PR Agent 将需求输入、资料调用、传播判断和内容输出组织成一条清晰流程。

01输入 brief填写项目背景、传播目标和初步需求。
02选择任务字段选择内容类型、品牌、语气和任务目标。
03检索相关资料从本地知识库中找到当前任务可引用的信息。
04生成策略骨架先整理传播逻辑，再展开具体文案内容。
05输出交付结果按结构展示结果，方便继续修改和导出。
Product Logic

三个模块，覆盖三类
内容任务

PR Agent 将内容生成、稿件处理和资料检索拆成独立模块，分别承接不同输入、处理逻辑和结果形式。

Module 01

传播内容生成

根据传播 brief 和参考资料，生成主题、方向、标题、导语、正文段落和社媒文案。输出先形成策略骨架，再展开具体内容。

Module 02

稿件审校改写

针对已有稿件完成审校、规范化和改写，检查事实、时间表述、品牌口径和 PR 语气，并输出可直接使用的修改建议或改写结果。

Module 03

知识库检索

将本地资料整理为可检索的上下文，为生成、审校和改写提供资料支撑，帮助输出更贴近项目背景与品牌表达。

显式字段内容类型、品牌、语气和任务目标以字段形式呈现，方便用户直接选择。
文档结果输出按章节整理，便于复制、审阅、导出和继续修改。
任务导向表达界面语言围绕任务、资料和交付展开，保持专业、克制和清晰。
Engineering Diagnosis

我修掉的一个 检索缺陷

这个项目不只是把页面做出来，也包括读懂系统为什么不稳定。

测试知识库检索时，我发现约三分之一的资料怎么搜都召不回。最后定位到原因：车型名过滤用了过宽的子串匹配，导致部分资料被错误排除。

≈35%知识库内容被静默漏掉，生成时拿不到应有上下文。
0修复后该类漏召回归零，相关检索能正常返回。
零跑回归检查，没有误伤其它车型资料。

Before / substring match

搜索 CLA 时，S-Class 里的 “cla” 也被命中，于是 S 级资料被误伤丢弃。

Incorrect exclusion

After / word boundary

改为识别完整车型词边界后，CLA 和 S-Class 各归各，不再互相污染。
